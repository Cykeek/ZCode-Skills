## Role Summary

> **Frontend Software Engineer** — Builds performant, accessible, delightful user interfaces. Owns the "last mile" of product experience. Expert in React ecosystem, TypeScript, modern CSS, testing, and performance. Collaborates deeply with Design, UX, and Backend. At Senior+ defines frontend architecture and design system strategy; at Staff+ sets org-wide platform direction.

---

## Core Responsibilities (ATS-Keyword-Rich Bullet Bank)

### UI Development (All Levels)
- Build **production-grade React components** with **TypeScript** (strict mode, generics, discriminated unions)
- Implement **responsive, accessible UIs** from **Figma** specs: **Tailwind CSS**, **CSS Modules**, **CSS-in-JS**
- Develop **server-rendered** and **statically generated** pages with **Next.js App Router** (RSC, Server Actions, Streaming, Suspense)
- Integrate **GraphQL** (Apollo, Urql, Codegen) and **REST** (TanStack Query, SWR) with **type-safe** data fetching
- Write **comprehensive tests**: **Vitest** (unit), **React Testing Library** (component), **Playwright** (E2E, visual regression)

### Performance & Quality (All Levels)
- Optimize **Core Web Vitals**: **LCP** <2.5s, **INP** <200ms, **CLS** <0.1, **TTFB** <800ms
- Implement **bundle optimization**: code splitting, tree shaking, dynamic imports, lazy loading
- Configure **image optimization**: Next/Image, AVIF/WebP, responsive, priority loading, blur placeholders
- Establish **performance budgets**; integrate **RUM** (Sentry, Datadog, Vercel Analytics) and **synthetic monitoring**
- Enforce **accessibility** (WCAG 2.1 AA): semantic HTML, ARIA, focus management, color contrast, screen reader testing

### Design Systems & Architecture (Senior+)
- Architect **design system component library**: monorepo (Turborepo/Nx), versioning, publishing, migration guides
- Build **design token pipeline**: Figma → Style Dictionary → CSS/TS/JS/Compose/iOS/Android
- Define **component APIs**: compound components, render props, slot patterns, polymorphic `as` prop
- Govern **design system**: RFC process, deprecation policy, adoption metrics, contribution model
- Lead **frontend architecture**: micro-frontends (Module Federation), plugin systems, shared kernel

### Technical Leadership (Staff+)
- Define **frontend platform strategy**: build vs. buy, framework selection, tooling investment, team structure
- Set **org-wide standards**: TypeScript config, ESLint rules, testing philosophy, performance budgets
- Lead **major migrations**: Pages→App Router, CSR→RSC, JavaScript→TypeScript, legacy→modern
- Mentor **Senior+ engineers**; grow **Staff+ engineers**; define **career ladder** and **hiring bar**
- Represent **frontend** in **executive reviews**, **board presentations**, **industry conferences**
