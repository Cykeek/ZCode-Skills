# Website Design — Deep Reference (Minimal & Calm, Dribbble-Inspired)

This is the deep reference for designing minimal, calm websites and landing pages inspired by the best Dribbble designers (2024-2026). Read this when designing websites, landing pages, marketing sites, product pages, or any web-based experience. **Unless the user explicitly requests a different visual style, default to minimal, calm, spacious design.**

Reference designers: Taras Migulko, Tran Mau Tri Tam, Outcrowd, Fireart Studio, RonDesignLab, Sergey Valiukh, Sasha Beliaev, Tubik Studio

---

## 1. The Minimal Website Design Philosophy

A minimal website doesn't feel empty — it feels intentional. Every element earns its place through purpose.

**Core principles**:
1. **Content is king** — The interface recedes so the message, product, or service can advance. Users should notice the content, not the container.
2. **Whitespace is the most powerful design tool** — Space signals importance, creates hierarchy, and communicates confidence. A design that can breathe feels premium.
3. **One primary action per view** — Every screen (especially above the fold) should have one clear thing the user should do. Everything else supports that action.
4. **Typography is 90% of web design** — On the web, people read (or scan). Type choice, size, weight, line-height, and measure determine whether the site feels professional or amateur.
5. **Performance is a design feature** — Minimal visual design must be matched by minimal code. Load time, image optimization, and smooth scrolling are UX requirements.

## 2. Landing Page & Website Design Process

### Step 1: Define the Goal

| Page type | Primary goal | Secondary goal |
|---|---|---|
| **Landing page** | Convert visitor (sign up, buy, download) | Educate about value proposition |
| **Product page** | Drive purchase or trial | Answer pre-purchase questions |
| **About page** | Build trust and connection | Communicate mission/values |
| **Blog/article** | Deliver readable content | Encourage subscription/engagement |
| **Portfolio/showcase** | Demonstrate quality of work | Generate leads/inquiries |
| **Documentation** | Enable self-service problem-solving | Reduce support tickets |

### Step 2: Structure the Content Hierarchy

**The proven landing page flow**:
```
Hero (Headline + Subhead + CTA + Visual)     — What? Why care? What to do?
Social proof (Logos, testimonials, stats)     — Who else trusts this?
Problem/Solution or How It Works              — What do you solve? How?
Features/Benefits (3 columns or alternating)  — What exactly do I get?
Proof (Case studies, detailed testimonials)   — Show it works for real people
Pricing (optional)                            — What does it cost?
FAQ                                           — Answer remaining objections
Final CTA (Last chance to convert)            — What's the next step?
Footer                                        — Links, legal, social
```

## 3. Dribbble-Inspired Visual Language for Websites

### 3.1 Default Color Palettes

**Palette 1: Light & Airy (default for most websites)**
```
Background:  #FAFAFA or #F8F9FA
Section alt: #F3F4F6 or #F0F2F5
Card:        #FFFFFF
Text primary: #111827 (near-black, not pure black)
Text secondary: #6B7280 (medium gray)
Text muted:   #9CA3AF (light gray)
Accent:       #4F6CF7 or #5680E9 (brand color)
Accent hover: darken accent by 10%
CTA text:     #FFFFFF (on accent)
Border/divider: #E5E7EB (1px, very subtle)
Footer bg:    #111827 or #1F2937
Footer text:  #D1D5DB
```

**Palette 2: Warm Neutral (for creative/brand sites)**
```
Background:  #F7F6F3
Section alt: #F0EEE9
Card:        #FFFFFF
Text primary: #2D2A24
Text secondary: #9C978E
Accent:       #D4785C (terracotta) or #4F6CF7
Border:       #E8E4DE
```

### 3.2 Typography for Websites

**Type scale (4pt base)**:
```
Display/Hero: 56-72px / 1.1 line-height / 700-800 weight / -0.02em tracking
H1:           40-48px / 1.15 / 700 / -0.01em
H2:           32-36px / 1.2 / 600-700 / 0
H3:           24-28px / 1.25 / 600 / 0
H4:           20-22px / 1.3 / 600 / 0
Body large:   18px / 1.6 / 400 / 0
Body:         16px / 1.6 / 400 / 0
Body small:   14px / 1.5 / 400 / +0.01em
Caption:      12-13px / 1.4 / 400 / +0.02em
```

**Recommended font pairings**:
| Heading | Body | Vibe |
|---|---|---|
| Inter Bold | Inter Regular | Clean, safe, modern |
| Satoshi Bold | Satoshi Regular | Slightly warmer, 2025 trendy |
| Plus Jakarta Sans Bold | Plus Jakarta Sans Regular | Rounded, friendly |
| Playfair Display Bold | Inter Regular | Editorial, sophisticated |
| Space Grotesk Bold | Inter Regular | Tech-forward, bold |

**Readability rules for web**:
- Line length (measure): 60-75 characters (optimal), 45-85 (acceptable)
- Line height: 1.5-1.7 for body text, 1.1-1.3 for headings
- Paragraph spacing: 1.5× line height
- Body size: never below 16px on desktop, 15px on mobile

### 3.3 Spacing & Layout for Websites

**Vertical rhythm scale**:
```
Section padding top/bottom:
  Desktop: 96-120px (large), 64-80px (standard), 48px (compact)
  Tablet:  80-96px (large), 56-64px (standard), 40px (compact)
  Mobile:  56-64px (large), 40-48px (standard), 32px (compact)

Content max-width: 1120-1200px (standard), 640-720px (reading), 1280-1440px (wide)
Content padding: 24px (mobile), 32px (tablet), 0-80px (desktop with max-width)

Between elements within a section: 24-32px
Between heading and body text: 16-20px
Between label and input/button: 8-12px
```

**Grid system**:
```
Desktop (1200+): 12 columns, gutter 24-32px
Tablet (768-1199): 8-12 columns, gutter 20-24px
Mobile (< 768): 4 columns, gutter 16-20px
```

### 3.4 Hero Section Design

The hero is the most critical section — it must communicate value in under 3 seconds.

**Hero layout patterns (from Dribbble 2024-2026)**:

| Pattern | Layout | Best for |
|---|---|---|
| **Centered hero** | Headline centered → subhead → CTA centered → visual below | Simple message, single audience |
| **Split hero** | Text left (50%), visual right (50%) | More text, product/feature showcase |
| **Visual-first hero** | Full-bleed visual/text overlay | Brand-driven, emotional appeal |
| **Headline-only hero** | Just a large headline + minimal sub + CTA | Ultra-minimal, confident brands |
| **Integrated hero** | Text interwoven with visual element | Creative/experimental brands |

**Centered hero (most common minimal pattern)**:
```
┌──────────────────────────────────────────────────┐
│  Logo  [Nav 1] [Nav 2] [Nav 3]  [CTA: Try Free]  │ ← header
│                                                    │
│                                                    │
│         ✦  Headline: One Line of Impact            │ ← 56-72px, bold
│         Subheadline that explains what you          │
│         do in one or two clear sentences.          │ ← 18-22px, secondary
│                                                    │
│              [Get Started →]  [Learn More]          │ ← CTAs
│                                                    │
│         ★ Trusted by 10,000+ companies ★            │ ← social proof
│                                                    │
│           ┌──────────────────────────┐              │
│           │     Hero Visual/Shot     │              │ ← product screenshot
│           │                          │              │    or illustration
│           └──────────────────────────┘              │
│                                                    │
└──────────────────────────────────────────────────┘
```

**Hero design rules**:
- Headline: single line when possible (max 2 lines). Clear, specific, benefit-focused
- Subheadline: 1-2 sentences explaining the value proposition. No jargon.
- Primary CTA: filled button, accent color, clear action text ("Get started", "Try free")
- Secondary CTA: outline or text link ("Learn more", "See how it works")
- Social proof: logos, testimonial quote, user count — positioned to support trust
- Hero visual: high-quality, purposeful. Never generic stock photography
- Spacing: 120-160px padding above and below the hero content (creates the "premium" feel)

### 3.6 Navigation Patterns

| Pattern | Usage | Best for |
|---|---|---|
| **Centered nav** | Logo center, links on either side | Portfolio, creative brands |
| **Left-aligned nav** | Logo left, links center, CTA right | Most SaaS, standard websites |
| **Hamburger nav** | Hidden menu, icon-only trigger | Minimal aesthetics, mobile-first |
| **Sticky header** | Nav fixed at top on scroll | Long pages, content-heavy sites |
| **Transparent header** | Nav overlays hero, becomes solid on scroll | Hero with full-bleed visual |

**Standard website navigation anatomy**:
```
Desktop (1200+):
┌──────────────────────────────────────────────────────┐
│  Logo/Wordmark    Link  Link  Link  Link  [CTA Button] │ ← sticky, 64-80px height
└──────────────────────────────────────────────────────┘
Links: 14-16px, medium weight, secondary color. Active: accent or bold.
CTA: filled button, 14-16px, comfortable padding 12-16px horizontal

Mobile (< 768):
┌──────────────────────────────────────────────────────┐
│  Logo/Wordmark                    [Menu ☰]            │ ← 56-64px height
└──────────────────────────────────────────────────────┘
Hamburger opens full-screen overlay or slide-in drawer
```

## 4. Website Design States

### 4.1 Loading / Skeleton (Content Pages)

- Show skeleton matching the layout: heading bar, text lines, image placeholder
- Shimmer animation: 1.5s ease-in-out
- No full-page spinners

### 4.2 404 / Error Page

- Large display numeral "404" or "Oops" in bold, 96-120px
- Short explanation: "This page doesn't exist" (14-16px, secondary)
- CTA: "Back to home" in accent button
- Gentle illustration (optional, 160-200px)
- No aggressive red, no "you broke it" messaging — calm, helpful, minimal

### 4.3 Empty Searches (Blogs, Products, Documentation)

- "No results for [query]" — 18px, semibold
- Suggestions: "Try different keywords" or show popular alternatives
- No angry illustrations, no red text — calm and helpful

### 4.4 Form States

- **Default**: Clean label + input with subtle border (#D1D5DB, 1px, 8px radius)
- **Focus**: Accent-colored border + subtle ring (box-shadow 0 0 0 3px rgba(79,108,247,0.1))
- **Error**: Red border + inline error message below in soft red (12-13px)
- **Success**: Green border + optional checkmark
- **Disabled**: 50% opacity, no pointer events
- **Placeholder**: 16px, muted text (#9CA3AF), sentence case

## 5. Responsive Website Patterns

### Desktop (1200px+)
- Multi-column layouts, full content
- Navigation visible, sticky header
- Full typography scale

### Tablet (768-1199px)
- 2-column grids collapse as needed
- Navigation may condense (hide less important links, show hamburger)
- Slightly smaller type (reduce hero to 40-48px)
- Sections stack vertically

### Mobile (< 768px)
- Single column layout
- Hamburger navigation
- Reduce hero: 32-36px headline, 16-18px body
- CTAs full-width (thumb-friendly, 48px+ height)
- Touch targets: 44px+ minimum
- Safe areas for notched devices

## 7. Website Content Guidelines

- **Headlines**: 5-9 words. Specific, benefit-driven. "Build faster with automated workflows" not "Our platform helps you build"
- **Subheadlines**: 10-20 words. Clarify the headline. "Connect your tools, automate repetitive tasks, and ship 3x faster."
- **Button text**: 2-4 words. Action + outcome. "Start free trial", "Get a demo", "See pricing"
- **Body text**: Short paragraphs (2-3 sentences max). Bullet points for features. No jargon. Readable at 8th-grade level.
- **Error messages**: "Something went wrong. Please try again." — calm, plain language, solution-oriented
- **404 pages**: "This page doesn't exist. Let's get you back on track." — helpful, not apologetic or frantic

## 8. Anti-Patterns (from Dribbble Analysis)

| Anti-pattern | Why it fails | Minimal fix |
|---|---|---|
| **Hero carousel/slider** | Users almost never see slide 2+. Kills conversion. | Single strong hero message. A/B test alternatives instead. |
| **Stock photography** | Looks generic, untrustworthy. Hurts credibility. | Custom illustrations, product screenshots, or no visual at all |
| **Auto-playing video with sound** | Aggressive, annoying, accessibility violation | Click-to-play or muted auto-play with clear controls |
| **Multiple competing CTAs** | Analysis paralysis. Users don't know what to do. | One primary CTA per view. Secondary as text link. |
| **Heavy textures/patterns** | Visually noisy, dates quickly | Clean solid backgrounds or subtle gradients |
| **Social media icons in header** | Visual noise, encourages leaving your site | Move to footer. Keep header focused on core navigation. |
